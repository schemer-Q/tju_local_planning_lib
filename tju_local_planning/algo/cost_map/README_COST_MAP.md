# 代价地图实现总览（映射与优先级）

本文档列出在本工程 `cost_map` 下优先实现的三种代价地图算法、实现难度及推荐用途。

- 优先级/难度映射：
  - 高优先级 / 低难度：栅格占据图（Occupancy Grid） — 推荐作为基础实现，用于城市道路与实时避障。
  - 中优先级 / 低难度：欧式距离变换（EDT） — 依赖占据图，作为距离/安全代价层，计算快速且必要。
  - 中优先级 / 中等难度：高度图 / 高度栅格（Elevation Grid） — 面向地面可遍历性评估，适用于叉车/越野场景。

- 文件布局（新增）：
  - `occupancy_grid.md`：栅格占据图实现笔记（优先实现）
  - `edt.md`：欧式距离变换实现笔记（依赖占据图）
  - `elevation_grid.md`：高度图实现笔记（地形可遍历性）

实现建议：
- 先实现增量更新的二维 `OccupancyGrid`（log‑odds），提供全域重建与增量更新 API；
- 在 `OccupancyGrid` 基础上实现快速的 EDT 模块（局部窗口或滑动窗口优先），并生成距离代价层；
- 高度图模块可并行实现，负责地面分割、每格高度统计与可遍历性打分；
- 保持模块解耦：输入采用上游点云转成的库内结构体（含坐标系与时间戳），提供 `update(point_cloud, pose)` 与 `query(x,y)` 等接口。
